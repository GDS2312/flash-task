# 翼记 YiNote — 产品介绍手册

> **中国电信天翼品牌 · 多模态 AI 任务助手**｜语音一句话、拍张照、写段笔记，瞬间变成结构化待办清单

---

## 一、产品概述

### 1.1 这是什么

**翼记** 是中国电信天翼品牌旗下的轻量级多模态 AI 任务管理工具。它解决一个高频痛点：

> 你在会议/路上/工位上，脑子里冒出要做的事，或者看到白板上的任务、收到一段语音——这些"念头"如何最快变成可追踪、可提醒的待办清单？

传统做法：打开备忘录 → 打字 → 复制到任务软件 → 设置提醒。三步以上，大多数人就放弃了。

翼记把这个链路缩短为**一步**：说话 / 拍照 / 粘贴文字 → AI 自动拆解为结构化任务 → 自动归类、识别日期优先级 → 到点提醒。

### 1.2 核心差异

| 对比维度 | 传统任务工具 | 翼记 |
|----------|------------|---------|
| **录入方式** | 手动打字 | 语音/图片/文字三模态 |
| **任务拆解** | 用户自己拆分 | AI 自动识别并拆解 |
| **日期识别** | 手动选日期 | 自然语言提取（"下周一前"→ 自动算日期） |
| **优先级** | 手动标记 | 语义识别（"紧急"→ 高优先级） |
| **标签分类** | 手动打标签 | 自动归类（"会议/开发/采购/文档..."） |
| **数据隐私** | 上传服务器 | 纯浏览器本地存储 |
| **价格** | 通常订阅制 | 完全免费 |

### 1.3 适用人群

- **团队管理者**：会议中听到的 Action Items 直接语音录入
- **项目经理**：白板上拍张照，任务直接进清单
- **研发人员**：随口说的 Bug 修复、代码优化不遗漏
- **任何想减少"遗忘成本"的人**

## 二、功能详解

### 2.1 语音录入 🎙️

**使用场景**：开车时想到的事、会议中不方便打字、走路时冒出的灵感

**操作流程**：
1. 点击右下角 `+` 按钮 → 选择「语音录入」
2. 点击「开始录音」，正常说话
3. 说完点击「停止录音」
4. 点击「AI 提取任务」→ 系统自动拆分并识别
5. 确认后「全部添加到列表」

**技术要求**：需使用 Chrome 浏览器（Web Speech API 支持最佳），首次使用会请求麦克风权限。语音数据不上传服务器，仅在浏览器本地处理。

**识别能力**：
- 自动拆分多个任务（通过标点、语气停顿）
- 识别日期（今天/明天/后天/下周一/X月X日）
- 识别优先级（紧急/尽快 → 高 / 不急 → 低）
- 自动打标签（会议/开发/文档/采购...）

---

### 2.2 图片识别 📷

**使用场景**：白板上的任务分工、手写便签、会议纪要拍照、聊天截图

**操作流程**：
1. 点击 `+` → 选择「图片识别」
2. 拖拽图片到上传区，或点击选择文件
3. 手动输入图片中的文字内容（规则引擎模式）
4. 或配置 LLM API 后使用「AI 识别图片内容」自动提取
5. 点击「AI 提取任务」拆解为任务清单
6. 确认后添加到列表

**两种模式**：
- **默认模式**：用户手动输入图片中的文字，AI 从文字中提取任务
- **LLM 模式**：在设置中配置 OpenAI 兼容 API 后，可直接识别图片内容（支持手写体、白板、文档截图）

---

### 2.3 文字输入 ⌨️

**使用场景**：从邮件/微信/钉钉复制一段话、手动键入多条待办

**操作流程**：
1. 点击 `+` → 选择「文字输入」
2. 粘贴或输入文字
3. 选择模式：
   - **AI 智能提取**：自动拆分、识别日期优先级标签
   - **直接添加**：作为单条任务直接加入列表（支持回车连续添加）
4. 确认任务列表

**输入示例**：
```
明天下午3点跟张工开会讨论AI平台方案，需要提前准备PPT。
周五前完成服务器采购，优先级很高。
记得联系李总确认下周出差安排。
```
AI 会自动拆解为 3 条独立任务，分别标记日期和优先级。

---

### 2.4 任务管理 📋

| 功能 | 说明 |
|------|------|
| **状态流转** | 待办 → 进行中 → 已完成（点击左侧圆圈切换） |
| **优先级** | 高（红色）/ 中（黄色）/ 低（灰色） |
| **截止日期** | 支持日期 + 时间，逾期红色提醒 |
| **标签分类** | AI 自动标签 + 手动编辑 |
| **搜索** | 标题/描述/标签全文搜索 |
| **筛选** | 按状态、优先级筛选 |
| **排序** | 按优先级/截止日期/创建时间排序 |

---

### 2.5 数据管理 💾

- **本地存储**：所有数据存储在浏览器 localStorage，不上传任何服务器
- **导出备份**：点击侧边栏「导出备份」→ 下载 JSON 文件
- **导入恢复**：点击「导入数据」→ 选择之前备份的 JSON 文件
- **跨设备**：目前不支持自动同步（隐私优先设计），可通过导出/导入手动迁移

---

### 2.6 通知提醒 🔔

- **浏览器通知**：在设置中开启后，任务到期时弹出桌面通知
- **提醒时间**：可配置提前 5分钟/15分钟/30分钟/1小时/1天 提醒
- **注意**：需浏览器允许通知权限；浏览器需在后台运行

## 三、AI 引擎说明

### 3.1 规则引擎（默认）

无需联网、无需 API Key，纯浏览器端运行。

**能力范围**：
- ✅ 识别 **18 类** 中文动作词（需要/必须/完成/提交/联系/采购/处理...）
- ✅ 提取日期（今天/明天/后天/下周X/X月X日/X天后）
- ✅ 判断优先级（紧急/加急→高 / 不急/有空→低）
- ✅ 自动分类 **12 类** 标签（会议/开发/设计/文档/采购/调研/客户/测试/学习/人事/运维/AI）

### 3.2 LLM 大模型（可选配置）

在设置中切换到 LLM 模式，配置 OpenAI 兼容 API：

| 配置项 | 示例值 |
|--------|--------|
| API 地址 | `https://api.openai.com/v1/chat/completions` |
| API Key | `sk-...` |
| 模型名称 | `gpt-4o-mini` |

**LLM 模式的额外能力**：
- ✅ 图片内容自动识别（手写、白板、文档截图）
- ✅ 更高精度的任务提取
- ✅ 复杂语境下的意图理解

**隐私说明**：LLM 模式下，文字/图片内容会发送到你配置的 API 端点。请确保 API 服务符合你所在组织的安全合规要求。

## 四、典型使用场景

### 场景 1：晨会快速记录
> 早会上大家七嘴八舌，打开语音录入 → 放在桌上 → 会议结束点「AI 提取」→ 所有人的 Action Items 自动进清单

### 场景 2：白板任务拍照
> 项目启动会画了满满一白板的任务分工 → 拍张照 → 手动录入关键任务 → AI 拆解为结构化清单 → 分享给团队

### 场景 3：邮件/消息转任务
> 领导微信发了 3 段语音交代工作 → 打开闪记任务 → 复制语音转文字结果 → AI 提取出 5 条任务 → 逐个跟踪

### 场景 4：个人 GTD
> 通勤路上突然想到 3 件事 → 语音录入 → 到公司打开电脑 → 任务已经在列表里（同一浏览器数据持久化）

## 六、快速开始

### 线上使用
直接访问：**https://gds2312.github.io/flash-task/**

推荐使用 Chrome 浏览器（语音识别效果最佳）。

### 本地部署
```bash
git clone https://github.com/GDS2312/flash-task.git
cd flash-task
npm install
npm run dev
```

### 开启 LLM 增强模式
1. 点击左侧「设置」
2. AI 引擎 → 选择「LLM 大模型」
3. 填入你的 OpenAI 兼容 API 信息
4. 保存 → 即可使用图片识别和更高精度的任务提取

## 七、常见问题

**Q: 我的数据安全吗？**
A: 规则引擎模式下，所有数据仅存储在你的浏览器本地，不上传任何服务器。LLM 模式下，文字内容会发送到你配置的 API 端点。

**Q: 支持哪些浏览器？**
A: Chrome/Edge 完整支持（含语音识别）。Safari/Firefox 支持文字和图片功能，语音识别效果较差。

**Q: 语音识别支持哪些语言？**
A: 当前配置为中文（zh-CN），可在代码中修改为其他语言。

**Q: 可以在手机上用吗？**
A: 支持移动端浏览器访问，响应式布局适配手机屏幕。

**Q: 如何备份数据？**
A: 点击侧边栏「导出备份」下载 JSON 文件，在新设备上「导入数据」即可恢复。

**Q: 能否团队共享？**
A: 当前版本为个人工具，不支持多用户协作。后续可基于此架构扩展后端。

**Q: 完全免费吗？**
A: 是的。规则引擎模式永久免费。LLM 模式需自备 API Key（可使用免费额度的服务）。
